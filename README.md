# 基于偶图匹配的课程项目任务分配系统

本项目将课程项目中的“小组成员-任务分工”问题建模为偶图匹配问题，并提供一个 Streamlit 交互界面，用于录入数据、展示算法推演过程、输出最终分工方案，并导出 Markdown 小组分工文件。

## 功能概览

- 支持无权最大匹配：使用匈牙利算法，让可分配任务数量最大。
- 支持最大权最优匹配：使用 KM 算法，让小组整体熟练度/意愿得分最大。
- 支持内置案例和自定义数据录入。
- 支持可视化查看交错树、匹配边、增广路和 KM 相等子图。
- 支持一键保存或下载小组分工导出文件。

## 项目结构

```text
.
├── app.py                 # Streamlit 交互界面
├── algorithms.py          # 匈牙利算法与 KM 算法实现
├── reporting.py           # 小组分工导出文本生成逻辑
├── sample_data.py         # 内置演示数据
├── requirements.txt       # Python 依赖
└── tests/                 # 算法与报告生成测试
```

## 运行方法

建议使用 Python 3.10 或更高版本。

```bash
pip install -r requirements.txt
streamlit run app.py
```

启动后浏览器会打开本地页面。如果没有自动打开，可以访问终端中显示的 `http://localhost:8501`。

## 使用说明

1. 在左侧控制台选择“无权最大匹配”或“最大权最优匹配”。
2. 在“数据建模”页选择内置案例，或自定义输入成员、任务、能力边/权值矩阵。
3. 在“算法推演”页拖动步骤条，观察匹配如何逐步形成。
4. 在“分工导出”页查看最终分工表。
5. 点击“保存分工到工作区”生成 `小组分工导出.md`。

注意：`小组分工导出.md` 是程序界面导出的分工结果；提交给课程作业的报告文件不纳入 GitHub 仓库。
